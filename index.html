Process

How RIPPER works

Five steps from project brief to actionable risk register. No prompting required — just ask.

1

Load project context

Drop the RIPPER folder into a Claude Project. The project brief, scoring rules, and identification patterns load automatically — no copy-pasting required.

2

Identify risks

RIPPER scans the project brief against 12 identification patterns across 6 risk categories: schedule, budget, scope, resource, technical, and external.

3

Score each risk

Every risk is scored on Probability (1–5) × Impact (1–5), with rationale grounded in specific project facts — not generic boilerplate.

4

Generate the register

A structured HTML risk register is produced — sorted critical first, with detail cards, mitigation recommendations, and contingency plans for every risk.

5

Refine interactively

Challenge a score, add a risk RIPPER missed, mark one as mitigated, or ask what-if questions. RIPPER updates the register through conversation.

Capabilities

Built to make decisions, not ask for them

Every design choice removes friction. RIPPER acts on clear intent — no hand-holding, no clarifying questions before it produces output.

🔍

Pattern-Based Identification

12 identification patterns detect risk across 6 categories. Each pattern maps to a score range, then calibrates against project-specific signals.

📊

Scored & Prioritized

Every risk gets a Probability × Impact score (1–25) with four priority tiers. The register sorts critical-first automatically.

💬

Interactive Refinement

Challenge a score, add a missed risk, mark one mitigated, or run what-if scenarios. The register updates in real time through conversation.

📄

HTML Output, Zero Dependencies

The register is a self-contained HTML file with inline styles. No external stylesheets, no frameworks. Open it in any browser and it works.

Coverage

Six categories. One specialist.

RIPPER identifies risks across every dimension of a project — not just the obvious schedule and budget concerns.

📅

Schedule

Hard deadlines, compressed timelines, sequential external dependencies, and milestones with no float or buffer.

💰

Budget

Thin contingency reserves, cost exposure from scope or timeline changes, and underfunded phases of work.

📐

Scope

Technically undefined deliverables, stakeholder misalignment, and elements with no agreed implementation approach.

👤

Resource

Single points of failure on people, shared resources across projects, and key-person dependencies with no backup.

⚙️

Technical

New technology adoption, legacy system integrations, undefined implementation approaches, and unknown complexity.

🌎

External

Regulatory approvals, vendor performance, customer-controlled processes, and events outside the project team’s control.
